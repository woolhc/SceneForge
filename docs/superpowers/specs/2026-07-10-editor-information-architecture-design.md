# 编辑器信息架构与双模式布局设计

## 目标

在不删除现有功能、不修改后端项目数据结构的前提下，重构编辑器的功能入口与空间布局，使其更接近剪映的使用逻辑：入口稳定、上下文明确、常用操作就近、高级能力可直接到达，同时支持专业编辑与轻量检查两种工作方式。

## 已确认决策

- 采用固定四区的剪映式上下文工作区。
- 默认进入专业模式。
- 简洁模式用于 AI 成片后的快速检查和轻量编辑。
- 现有功能全部保留并提供直接入口，不放入“更多”或隐藏菜单。
- “AI 一键成片”作为顶部左侧固定主入口。
- 简洁模式与专业模式共享项目、选择、播放、历史和保存状态。

## 当前问题

### 信息架构

- 左侧仅按“媒体、文本、音频、转场”分组，混合了内容添加、AI 生成、字幕样式和对象编辑等不同任务。
- 转场拥有一级入口，但字幕、特效等同等级能力分散在文本面板或右侧属性面板中，层级不一致。
- AI 一键成片主要依赖首页入口，进入编辑器后不够突出。
- 部分功能在顶部、时间线工具栏、右键菜单中重复出现，但没有明确的主入口。

### 属性编辑

- 右侧属性面板将字幕、时长、音频、变速、滤镜、变换、关键帧、蒙版和特效纵向堆叠。
- 属性组织主要由实现顺序决定，而不是由选中对象和编辑任务决定。
- 低频高级参数会把高频基础参数挤出首屏，用户需要长距离滚动。
- 部分属性仅在特定轨道有效，但面板缺少稳定的分类导航和对象摘要。

### 操作路径

- 删除、复制、导出等操作存在多个同级可见入口，增加视觉噪声。
- 时间线工具栏混入“设置封面”和“导出”等非时间编排操作。
- 预览诊断入口与普通缩放控制并列，对普通用户过于突出。
- 选中对象后，左侧工具和右侧属性之间缺少明确联动。

### 可维护性

- `src/App.tsx` 同时承担数据编排、命令处理、布局和大部分属性表单渲染，文件已超过 4600 行。
- 属性区缺少可独立测试的面板边界，继续增加功能会进一步放大入口混乱。

## 总体布局

编辑器使用五个稳定区域，其中中上部四区构成主要工作区：

1. **顶部全局栏**：项目级操作和模式切换。
2. **左侧功能导航**：添加内容和调用创作能力。
3. **中间预览区**：画布、播放控制和画布内直接编辑。
4. **右侧上下文属性区**：修改当前选中对象。
5. **底部时间线**：轨道、片段和时间编排。

空间职责保持单一：左侧负责“添加”，右侧负责“修改”，底部负责“编排”，顶部负责“项目”，中间负责“查看与直接操作”。

## 顶部全局栏

### 左侧

- 返回首页。
- 固定主按钮“AI 一键成片”。
- 项目切换器。

### 中部

- 项目名称。
- 画幅比例。
- 撤销、重做和保存状态。

### 右侧

- 专业模式/简洁模式切换。
- 设置。
- 导出主按钮。

FFmpeg 等运行环境状态不持续占用顶部空间。正常时移入状态栏；异常时通过可操作提示展示。

## 左侧功能导航

左侧使用窄图标导航栏加内容面板，一级入口固定为：

1. **媒体**：本地素材、项目素材、在线视频和图片搜索。
2. **文本**：普通文本、文案和文本模板。
3. **音频**：音乐、音效、配音生成和音色管理。
4. **字幕**：字幕识别、翻译、手动字幕、SRT 导入和字幕预设。
5. **转场**：入场/出场转场及持续时间。
6. **特效**：滤镜、视觉特效和蒙版预设的添加入口。

左侧面板只负责创建、搜索、选择和添加能力。已经添加到时间线的对象，其详细参数统一在右侧编辑。

### 现有入口迁移

- `MediaPanel` 保留在“媒体”。
- `TextPanel` 拆分为“文本”和“字幕”两个面板。
- `AudioPanel` 保留在“音频”，内部区分配音、音乐/音效和音色。
- `TransitionPanel` 保留在“转场”。
- 右侧现有滤镜、视觉特效和蒙版预设增加“特效”添加入口，但具体参数仍由右侧上下文属性区负责。

## 中间预览区

- 顶部仅显示画布名称、适配缩放和安全框/网格开关。
- 预览诊断移入开发者设置或诊断弹层，不与普通缩放控件同级常驻。
- 播放控制保留跳转、逐帧、播放、进度和时间码。
- 字幕、画中画和变换继续支持画布内直接拖动、缩放和旋转。
- 画布直接编辑时，右侧自动切换到与操作对应的属性分类，但不抢夺输入焦点。

## 右侧上下文属性区

### 固定头部

- 显示选中对象类型、名称、所属轨道和多选数量。
- 多选时只展示可批量修改的共同属性。
- 无选中对象时展示简短引导和最近使用操作，不显示空白长面板。

### 分类标签

标签根据对象类型动态启用，但标签位置保持稳定：

- **基础**：文案、时长、速度、素材替换、裁剪和常用开关。
- **画面**：位置、缩放、旋转、透明度、适配、滤镜、调色和蒙版。
- **动画**：关键帧、缓动、入场/出场动画、转场摘要和视觉特效动态参数。
- **音频**：音量、淡入淡出、降噪、声道相关能力和配音操作。
- **字幕样式**：仅字幕对象显示，包含字体、字号、颜色、描边、位置、卡拉 OK 和字幕动画。

所有现有功能仍直接可见：用户切换分类标签即可到达，不需要打开“高级”或“更多”。每个分类内部允许使用少量语义折叠组，但首次进入时常用组默认展开。

### 上下文联动

- 选择字幕片段时默认进入“字幕样式”。
- 选择音频或配音片段时默认进入“音频”。
- 选择视频或图片片段时默认进入“基础”，画布变换后自动切到“画面”。
- 点击关键帧标记时自动进入“动画”。
- 从左侧应用转场后，右侧进入“动画”并定位转场摘要。
- 用户手动切换过分类后，同类对象在当前会话中记住上次分类。

## 底部时间线

### 主工具栏

保留与时间编排直接相关的操作：

- 分割。
- 删除。
- 复制、粘贴和复制片段。
- 添加轨道。
- 添加章节。
- 吸附、联动选择等时间线行为开关。
- 时间线缩放。

### 迁移操作

- “导出”只保留顶部主入口，时间线不再重复显示。
- “设为封面”迁移到预览区画布菜单或项目操作区。
- 片段专属操作继续保留在右键菜单，并与右侧属性区形成快捷入口与完整入口的关系。

## 双模式行为

### 专业模式

- 默认模式。
- 展示完整左侧导航、右侧分类属性和多轨时间线。
- 所有功能直接可达。

### 简洁模式

- 复用同一工作区，不切换到另一套页面。
- 左侧默认收窄，仅保留图标导航。
- 右侧只显示当前对象的常用属性，但每个分类入口仍可直接切换。
- 时间线降低默认高度并隐藏轨道管理细节，保留播放、选择、裁剪、分割和删除。
- 用户展开任一高级区域时不修改项目模式；切回专业模式后恢复原面板尺寸。

### 状态规则

- 模式是 UI 偏好，不写入后端项目模型。
- 每次进入编辑器均默认使用专业模式。
- 简洁模式只在当前编辑会话中生效，不跨项目或重启持久化。
- 两种模式共享当前项目、当前选中对象、播放头、撤销栈和未保存更改。

## 组件边界

为避免继续扩大 `App.tsx`，布局改造同步拆分以下组件：

- `EditorTopbar`：全局项目操作和模式切换。
- `ToolRail`：左侧一级导航。
- `ToolPanel`：当前左侧功能面板容器。
- `PreviewWorkspace`：预览工具栏、画布和播放控制。
- `InspectorPanel`：对象摘要、分类标签和内容路由。
- `InspectorSection`：分类内的语义分组。
- `TimelineToolbar`：时间线专属操作。
- `EditorWorkspace`：可调整尺寸的整体布局。

业务命令和项目状态继续由现有 store 与 `App` 编排，第一阶段不进行大规模状态管理重写。拆分组件优先接收明确 props，避免新增全局依赖。

## 可用性与无障碍

- 所有仅图标入口必须有中文 `title` 与 `aria-label`。
- 当前一级导航和属性分类必须提供明显选中态。
- 键盘焦点不能因自动分类切换而丢失。
- 面板最小宽度下，文字标签可退化为图标，但功能不可消失。
- 模式切换、面板尺寸和标签选择不进入项目撤销历史。
- 危险操作继续要求明确反馈，并支持现有撤销能力时优先撤销而非二次确认。

## 错误与状态反馈

- AI 生成、字幕识别、配音和素材搜索的进行中状态显示在触发入口附近。
- 全局错误使用 Toast，并提供“打开设置”或“重试”等动作。
- 长任务不通过禁用整个编辑器阻塞普通剪辑操作，只有冲突命令局部禁用。
- 保存状态在顶部以“保存中/已保存/保存失败”表达，不依赖用户反复点击保存按钮确认。

## 测试策略

### 组件测试

- 不同轨道类型对应正确的默认属性分类。
- 模式切换不改变项目和选择状态。
- 左侧六个入口均可到达对应面板。
- 时间线工具栏不再重复显示导出。
- 多选时只显示共同属性和批量操作。

### 回归测试

- 现有媒体导入、字幕识别、配音、转场、滤镜、关键帧、蒙版、特效和导出入口仍可触发原命令。
- 画布拖动、缩放、旋转与右侧属性保持同步。
- 撤销、重做、保存和跨项目历史隔离不受布局拆分影响。

### 视觉验证

- 在 1280×720、1440×900 和 1920×1080 三种窗口尺寸下验证面板可用性。
- 验证专业/简洁模式、无选中、视频、字幕、音频和多选状态。
- 检查顶部、左侧、右侧和时间线不存在横向溢出或关键按钮遮挡。

## 实施阶段

### 第一阶段：结构与入口

- 建立双模式 UI 状态。
- 拆分工作区骨架组件。
- 重组顶部、左侧导航、右侧分类和时间线工具栏。
- 保持现有业务回调和数据结构不变。

### 第二阶段：属性面板模块化

- 将当前长属性面板拆分为对象类型与属性分类组件。
- 建立分类默认值和上下文联动。
- 补齐多选共同属性规则。

### 第三阶段：视觉与响应式精修

- 统一尺寸、间距、选中态和控件密度。
- 完成简洁模式的尺寸策略。
- 验证常见窗口尺寸和键盘可用性。

## 非目标

- 本次不修改 Rust 后端命令或 FFmpeg 渲染逻辑。
- 本次不更改项目持久化格式。
- 本次不删除现有功能。
- 本次不引入新的 UI 组件依赖。
- 本次不追求一次性复刻剪映全部视觉细节。

## 完成标准

- 所有现有编辑能力均有清晰、稳定、直接可达的入口。
- 用户可以根据“添加、查看、修改、编排、项目操作”预测功能所在区域。
- 右侧属性不再依赖单一超长滚动页面。
- 默认专业模式与简洁模式切换不丢失编辑上下文。
- 相关 TypeScript 测试、构建和现有验证脚本通过。
