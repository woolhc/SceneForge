# SceneForge 应用图标与品牌更名设计

## 目标

将现有通用的“播放按钮”图标替换为具有独立品牌识别度的 SceneForge 图标，并将用户可见品牌从 SceneScript Desktop 统一更新为 SceneForge。改动必须保持现有项目、设置和用户数据兼容。

## 品牌定位

SceneForge 是面向专业视频创作者的桌面编辑与 AI 辅助创作工具。品牌需要同时表达三个概念：

- **Scene**：场景、镜头和叙事单元。
- **Forge**：将素材、脚本和创意锻造成完整作品。
- **Editing**：时间线切片、拼接和成片过程。

图标不以摄像机、播放键、场记板或 AI 星芒作为主体，避免落入同类视频工具的通用视觉模板。

## 图标概念

最终采用“锻造之 S”方案。图形由三段时间线切片构成抽象字母 `S`：

- 上段使用暖白色，代表脚本、素材和创作起点。
- 中段使用金色，代表锻造过程、时间线核心和成片高光。
- 下段使用薄荷绿色，代表创造、智能能力和最终输出。
- 三段之间保留明确切口，使图形同时具有剪辑时间线和熔铸拼接的语义。

字母 `S` 对应 Scene，切片的组合过程对应 Forge。图形应优先体现品牌字标，而不是直接说明“这是一个视频播放器”。

## 视觉规范

### 构图

- 使用正方形 SVG 母版，设计坐标为 `1024 × 1024`。
- 背景为深黑圆角方形，基础色为 `#111416`。
- 主体图形占画布宽高约 58%，四周至少保留 18% 安全边距。
- 主体视觉中心略高于几何中心，以补偿下段图形的视觉重量。
- 图形仅使用硬边和纯色，不添加描边、投影、纹理、玻璃效果或细碎高光。
- SVG 母版保留圆角背景；平台构建产物按 Tauri 的图标生成规则输出透明边缘和系统所需尺寸。

### 色彩

| 用途 | 色值 |
| --- | --- |
| 深色背景 | `#111416` |
| 薄荷绿 | `#22D3A6` |
| 金色 | `#F3C969` |
| 暖白 | `#F2F0EA` |

不引入额外品牌主色。小尺寸版本保持相同配色，不使用单独的简化色板。

### 小尺寸规则

- `16px` 和 `32px` 下仍须能辨认连续的 `S` 轮廓。
- 删除装饰性星芒等非主体元素，避免小尺寸噪点。
- 三段主体之间的最小间隔不得在缩放后消失或粘连。
- 不在图标中放置 `SceneForge` 文字或 `SF` 字母组合。

## 输出资产

以 `src-tauri/icons/icon.svg` 作为唯一可编辑母版，并由该母版生成以下平台资产：

- `icon.png`，至少保留 `1024 × 1024` 高分辨率版本。
- Tauri/macOS 所需的 `.icns` 和 PNG 尺寸集合。
- Tauri/Windows 所需的 `.ico` 和 PNG 尺寸集合。
- Linux 和通用桌面包所需的 PNG 尺寸集合。

生成资产必须来自同一 SVG，不允许手工维护造型不同的多套图标。

## 品牌更名范围

以下用户可见内容统一改为 `SceneForge`：

- Tauri `productName` 和窗口标题。
- Web 页面标题。
- 首页品牌名称。
- README 主标题、图片替代文本和当前项目级说明文档中的产品名称。
- 面向用户的错误对话框标题和运行错误文本。
- npm 包的展示名称可更新为 `sceneforge-desktop`，但不得影响持久化数据位置。

历史设计规范和实施计划属于决策记录，不批量重写其中的 SceneScript 名称，以保留历史上下文。

## 兼容边界

本次不修改以下内部标识：

- Tauri bundle identifier：`com.scenescript.desktop`。
- Rust crate 名和 Rust 模块引用。
- `SceneScriptDesktop` 应用数据目录。
- `scenescript.sqlite3` 数据库文件名。
- 现有 `localStorage` 键和功能开关键。
- 临时目录、测试临时文件前缀及现有缓存路径。

这些名称属于兼容接口，不是用户可见品牌。保留它们可避免升级后出现项目、设置、缓存或功能开关丢失。未来若需要彻底迁移，必须单独设计带回滚能力的数据迁移方案。

## 验证方案

### 视觉验证

- 导出并检查 `16、32、64、128、256、512、1024px` 图标。
- 检查轮廓是否清晰、颜色是否粘连、圆角和透明边缘是否正常。
- 检查 macOS Dock、Finder 和应用切换器中的圆角遮罩效果。
- 检查 Windows 桌面、任务栏和安装包图标。
- 将新旧图标并排比较，确认新图标不再依赖通用播放按钮语义。

### 工程验证

- 确认 Tauri 配置可以识别全部生成资产。
- 运行前端测试和生产构建。
- 运行 Tauri 配置或构建检查，确保产品名更新没有破坏打包。
- 搜索用户可见代码中的 `SceneScript` 残留，并区分历史文档与兼容标识。
- 检查 Git diff，确保没有意外修改数据路径、bundle identifier 或存储键。

## 完成标准

- SceneForge 图标在 `16px` 至 `1024px` 均保持清晰可辨。
- 图标与现有深色、薄荷绿、金色界面体系一致。
- 用户可见品牌统一显示为 SceneForge。
- 已有项目、设置和本地数据在升级后保持可访问。
- 前端构建与相关 Tauri 检查通过。
